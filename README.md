# STM32F103C8T6 智能小车项目 (by 404elf)

## 项目简介

本项目是一款基于 **STM32F103C8T6** 的入门级智能小车。

*   开发环境为：Keil uVision5，调试器选择为 ST-Link。
*   代码基于 **B站UP主“好家伙VCC”的教程视频**：[STM32智能小车V1.5](https://www.bilibili.com/video/BV1SY411L7rJ/) 进行了全方面的修改。
*   利用 **时间戳和状态机** 重构了业务逻辑，并 **参考 B站UP主“江协”** 的模块化思想优化了文件架构与代码管理。

*   芯片与蓝牙模块连接的 RX、TX 引脚也发生了损坏，因此程序和接线都做了相应调整。



小车支持以下操作模式：

*   **MODE0: 停止**
*   **MODE1: 超声波跟随**
*   **MODE2: 蓝牙遥控**
*   **MODE3: 超声波避障**
*   **MODE4: 普通循迹**
*   **MODE5: PID循迹** (当前PID参数待调试)

## 功能介绍

### MODE0: 停止

停止模式下，小车立即关闭左右电机输出，并将超声波舵机回正到 90°。
该模式既可作为上电后的默认安全状态，也可作为运行过程中的紧急停机状态。

### MODE1: 超声波跟随

小车通过 HC-SR04 持续测量前方距离，并根据距离远近自动前进、后退或停止。

*   当前方目标距离约为 20cm ~ 30cm 时，小车低速前进跟随。
*   当前方目标距离大于 30cm 时，小车适当提高速度靠近目标。
*   当目标距离小于约 15cm 时，小车主动后退，避免贴得过近。
*   中间安全距离区间内保持停止，以减少抖动。

代码中对距离数据做了一阶滤波处理，用于减小超声波测距波动带来的误动作。

### MODE2: 蓝牙遥控

通过 HC-05 蓝牙串口模块接收手机或上位机指令，实现手动遥控。

支持的方向控制如下：

*   `A`：前进
*   `B`：后退
*   `C`：右转
*   `D`：左转
*   `E`：停止

支持的模式切换如下：

*   `F`：切换到 MODE1 超声波跟随
*   `G`：切换到 MODE2 蓝牙遥控
*   `H`：切换到 MODE3 超声波避障
*   `I`：切换到 MODE4 普通循迹
*   `J`：切换到 MODE5 PID 循迹
*   `K`：切换到 MODE0 停止

若 15 秒内未继续接收到蓝牙控制数据，小车会自动停止，防止失控。

### MODE3: 超声波避障

该模式下小车会边前进边检测前方障碍物，必要时通过舵机带动超声波模块左右扫描后决定绕行方向。

其核心逻辑如下：

*   默认舵机朝前，前方无障碍时持续前进。
*   当检测到前方距离小于约 30cm 时，小车先停止并进入扫描流程。
*   先向右探测，再向左探测，优先选择有空余空间的一侧转向。
*   若左右都存在障碍，则先后退一段时间，再执行转向脱困。

针对超声波在斜面或边角环境下可能出现的远距离误判，代码中加入了异常值修正逻辑，以降低误判导致的撞击风险。

### MODE4: 普通循迹

普通循迹模式使用 4 路 TCRT5000 红外对管进行黑线检测，并根据组合状态进行差速转向。

当前实现特点如下：

*   支持直行状态识别。
*   支持轻微左转、轻微右转修正。
*   对黑色十字路口状态进行了单独保留。
*   默认按最近一次有效状态持续运动。

### MODE5: PID循迹

项目已经搭建了 PID 循迹的基础框架，包括：

*   PID 参数结构体与初始化函数。
*   基于 4 路红外状态的误差量化。
*   定时器中断周期计算 PID 输出。
*   根据 PID 输出对左右电机进行差速调节。

但因为还没有实地调参， PID 参数默认设置为 0，这点需要注意。

### 通用辅助功能

除上述模式外，项目还提供了以下辅助能力：

*   **OLED 实时显示：** 周期显示当前模式、超声波测得距离以及 ADC 采样换算后的电压值。
*   **串口调试输出：** 与 OLED 同步输出模式、距离和电压信息，便于串口助手观察运行状态。
*   **按键切换模式：** `KEY1` 用于在各模式之间循环切换，`KEY2` 用于立即切回停止模式，同时翻转 LED 状态作为提示。
*   **电池电压采样：** 通过 ADC 获取电压信息，可用于供电状态监测。
*   **舵机联动超声波：** 避障模式下舵机可调整超声波朝向，实现简易环境扫描。



## 文件架构

项目采用分层设计：

*   **Start/ ：** 启动文件。
*   **Library/ ：** 标准库函数。
*   **Algorithm/ ：** 算法层，目前主要包含PID算法。
*   **Hardware/ ：** 硬件层，包含各硬件的驱动。
*   **System/ ：** 系统层，负责全局宏定义与变量。
*   **User/ ：** 执行层，包含初始化与业务逻辑。
